Question 3

There are some rules for how you can name variables in JavaScript. What are they?

  1. It must begin with a letter, dollar sign, or an underscore. It cannot start with a number
  2. It can contain letters, numbers, dollar signs, or underscores. You cannot use a dash or period.
  3. It cannot contain keywords or reserved words
  4. Variable names are case sensitve so newItem and NewItem would be different variable names.
  5. Use names that describe the information that the variable stores.
  6. If the variable name is more than one word use camelCase or an underscore between the words.
Question 4

What is 'camelCase'?

It is using an uppercase letter to begin any word after the first word of a phrase. For example: newVariableName
